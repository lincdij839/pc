# PC Language - 多系统互联能力展示

## 🎯 互联后能做什么

PC Language 成功实现了 **5 个系统的互联**：
- 本机 Ubuntu (宿主机)
- Debian 容器
- Fedora 容器
- Arch Linux 容器
- Kali Linux 容器

---

## ✅ 已实现的功能

### 1. 📊 系统信息查询 (`multi_system.pc`)
- 查看所有系统的发行版信息
- 获取内核版本
- 查看 IP 地址
- 监控内存使用
- 统计已安装软件包数量
- 查看进程数

**运行：** `./zig-out/bin/pc multi_system.pc`

---

### 2. ⚡ 批量命令执行 (`batch_command.pc`)
- 在所有系统上执行相同命令
- 统一收集执行结果
- 支持任意 shell 命令

**示例：** 查看所有系统的磁盘使用情况

**运行：** `./zig-out/bin/pc batch_command.pc`

---

### 3. 📝 配置文件同步 (`sync_config.pc`)
- 创建统一的配置文件
- 同步到所有系统
- 自动应用配置（添加到 .bashrc）
- 支持环境变量、别名等配置

**功能：**
- 统一编辑器设置
- 统一语言环境
- 统一命令别名
- 跨发行版的更新命令

**运行：** `./zig-out/bin/pc sync_config.pc`

---

### 4. 💾 系统备份 (`backup_system.pc`)
- 备份所有系统的配置文件
- 支持从容器拉取文件
- 集中存储备份
- 便于恢复和迁移

**备份内容：**
- .bashrc 配置
- hostname 配置
- 其他重要配置文件

**运行：** `./zig-out/bin/pc backup_system.pc`

---

### 5. 📊 集群监控 (`monitor_cluster.pc`)
- 实时监控所有系统状态
- CPU 使用率
- 内存使用率
- 磁盘使用率
- 进程数统计
- 系统运行时间

**运行：** `./zig-out/bin/pc monitor_cluster.pc`

---

### 6. 📦 批量软件安装 (`batch_install.pc`)
- 在所有系统上安装相同软件
- 自动适配不同包管理器：
  - Ubuntu/Debian/Kali: apt
  - Fedora: dnf
  - Arch: pacman
- 检测是否已安装
- 统一安装流程

**运行：** `./zig-out/bin/pc batch_install.pc`

---

## 🚀 核心优势

### 1. 跨发行版统一管理
- 一套代码管理多个不同的 Linux 发行版
- 自动适配不同的包管理器
- 统一的命令接口

### 2. 批量操作
- 一次操作影响所有系统
- 大幅提高运维效率
- 减少重复工作

### 3. 配置同步
- 保持所有系统配置一致
- 便于团队协作
- 简化系统迁移

### 4. 集中监控
- 一个界面查看所有系统状态
- 快速发现问题
- 便于资源规划

### 5. 自动化备份
- 定期备份重要配置
- 快速恢复系统
- 防止配置丢失

## 📋 实际应用场景

### 场景 1: 开发环境统一
在所有系统上安装相同的开发工具，保持开发环境一致。

统一管理所有系统的配置文件，如 SSH 配置、Git 配置等。

实时监控所有系统的资源使用情况，及时发现性能问题。

### 场景 4: 批量部署
在所有系统上部署应用程序或更新配置。

### 场景 5: 灾难恢复
定期备份所有系统配置，快速恢复故障系统。
